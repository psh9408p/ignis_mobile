# Cafe24 상품 상세 커스텀 옵션 UI

Cafe24 상품 상세페이지의 기본 옵션 선택 영역을 골라담기 형태의 커스텀 옵션 UI로 확장한 과제입니다.  
기본 Cafe24 옵션 선택, 선택상품 목록, 수량 변경, 총금액 계산, 장바구니/바로구매 흐름은 유지하면서 사용자에게 더 명확한 옵션 선택 경험을 제공합니다.

## 구현 요약

### 1. 구매 옵션 바텀시트

- `구매하기`, `장바구니`, 하단 `바로구매하기` 클릭 시 기존 중앙 모달 대신 옵션 바텀시트를 엽니다.
- 바텀시트는 닫아도 제거하지 않고 숨기므로, 다시 열었을 때 선택한 옵션과 수량이 유지됩니다.
- 배경 클릭과 닫기 버튼으로 바텀시트를 닫을 수 있습니다.
- `0~1024px`에서는 고정 구매 영역과 바텀시트가 화면 전체 너비를 사용합니다.

### 2. Cafe24 기본 옵션과 동기화

- Cafe24가 렌더링한 기본 옵션 데이터를 읽어 라디오형 커스텀 옵션 목록으로 표시합니다.
- 커스텀 옵션 클릭 시 원본 Cafe24 옵션 버튼의 클릭 이벤트를 실행합니다.
- 선택상품 목록은 Cafe24의 `#totalProducts` 영역을 사용합니다.
- 수량 변경, 삭제, 총금액 계산, 장바구니 담기, 바로 구매는 Cafe24 기본 구매 로직을 사용합니다.

### 3. 중복 선택 및 추가 가능 횟수

- 옵션값 suffix를 기준으로 같은 개입수의 추가 가능 횟수를 제한합니다.
- 예를 들어 `30개입_1`, `30개입_2`가 등록되어 있으면 고객에게는 `30개입`으로 보이며, 최대 2회까지 선택할 수 있습니다.
- 첫 번째 선택은 내부적으로 `_1`, 두 번째 선택은 `_2`를 사용합니다.
- 최대 횟수에 도달하면 `2 / 2 선택`으로 표시되고 해당 옵션은 비활성화됩니다.
- 선택상품을 삭제하면 사용 가능한 슬롯이 다시 열립니다.

### 4. 외부 설정 JS

변경 가능한 옵션 표시 데이터는 [`js/module/product/option-picker-config.js`](js/module/product/option-picker-config.js)에서 관리합니다.

```js
window.OptionPickerConfig = {
  options: {
    '30개입': {
      badge: '인기',
      recommendation: '가장 많이 선택하는 구성입니다.',
      price: '(+2,000원)',
      maxSelectable: 2
    }
  }
};
```

설정 항목:

- `badge`: 강조 뱃지 문구
- `description`: 옵션 설명 문구
- `recommendation`: 추천 문구
- `price`: 고객에게 표시할 가격/추가금액 문구
- `maxSelectable`: 옵션별 최대 선택 횟수

Cafe24 기본 데이터가 기본값으로 사용되며, 외부 설정값이 있으면 표시 영역에 우선 적용됩니다. 실제 결제 금액과 상품 생성은 항상 Cafe24 기본 옵션 데이터를 기준으로 처리합니다.

### 5. 배송 조건 안내

- 바텀시트 하단에서 무료배송 기준 금액과 남은 금액을 실시간으로 안내합니다.
- 현재 무료배송 기준은 `40,000원`이며, 기준 금액 충족 시 무료배송 적용 상태를 표시합니다.

### 6. 선택 상태 표시

- 옵션별 선택 횟수를 `현재 선택 수 / 최대 선택 수`로 표시합니다.
- 최대 선택 횟수에 도달한 옵션은 비활성화 상태로 표시합니다.
- 선택된 상품은 별도 카드로 표시하고, 수량 조절과 삭제 버튼을 제공합니다.

### 7. 반응형 대응

- `0~1024px`: 모바일/태블릿 구매 UI와 바텀시트가 전체 너비로 동작합니다.
- `1025px 이상`: 기존 데스크톱 상품 상세 레이아웃을 유지합니다.
- 선택 옵션 카드, 옵션 목록, 하단 구매 영역이 화면 너비에 맞게 배치됩니다.

## 구현 설명

### 기본 구조

커스텀 옵션 UI는 Cafe24 기본 옵션 기능을 대체하지 않습니다. 기존 옵션 선택 버튼 위에 고객용 표시 레이어를 추가하고, 고객이 커스텀 옵션을 선택하면 해당하는 원본 Cafe24 옵션 버튼을 호출하는 방식으로 구현했습니다.

- `product/layer_option.html`에서 Cafe24 옵션 DOM을 분석합니다.
- 분석한 옵션값을 기준으로 커스텀 옵션 목록을 생성합니다.
- 커스텀 버튼 클릭 시 원본 Cafe24 옵션 버튼을 실행합니다.
- 이후 선택상품 목록, 수량 변경, 총금액 계산, 장바구니/바로구매는 Cafe24 기본 흐름을 따릅니다.

### 옵션값 처리 방식

Cafe24 옵션값은 `30개입_1`, `30개입_2`처럼 suffix가 붙은 형태를 기준으로 처리합니다.

- 고객 화면에는 suffix를 숨긴 `30개입`만 표시합니다.
- 같은 개입수의 suffix 개수로 최대 추가 가능 횟수를 판단합니다.
- 이미 선택한 suffix는 다시 선택하지 않고, 다음 사용 가능한 suffix를 선택합니다.
- 외부 설정의 `maxSelectable` 값은 실제 Cafe24에 등록된 suffix 슬롯 수보다 클 수 없습니다.

### 상태 동기화 방식

커스텀 UI와 Cafe24 선택상품 목록이 분리되지 않도록 다음 상태를 지속해서 동기화합니다.

- 원본 Cafe24 옵션 클릭으로 선택상품을 생성합니다.
- `#totalProducts`를 읽어 현재 선택된 원본 옵션과 개수를 계산합니다.
- 커스텀 옵션의 선택 횟수와 비활성 상태를 갱신합니다.
- 삭제 후에도 남은 선택 상태를 다시 계산하여 사용 가능한 옵션을 복구합니다.

### 스타일 구현

기존 상품 상세 레이아웃을 유지하면서 옵션 선택과 구매 흐름에 필요한 영역만 별도 스타일로 구성했습니다.

- 하단 고정 구매 버튼과 옵션 바텀시트
- 라디오형 커스텀 옵션 목록
- 선택 옵션 카드, 수량 컨트롤, 삭제 버튼
- 무료배송 조건 안내와 총금액 영역
- 모바일/태블릿 전체 너비 반응형 처리

### QA 중 확인한 보완 사항

- 비회원의 바로 구매는 Cafe24 정책에 따라 로그인 화면으로 이동할 수 있습니다.
- 배송 조건 안내는 고객용 UI 안내이며, 실제 배송비 최종 계산은 Cafe24 주문 정책을 따릅니다.
- Cafe24 관리자 옵션 데이터와 외부 설정의 `maxSelectable` 값이 다르면, 실제 등록된 suffix 슬롯 수를 초과해 선택할 수 없습니다.

## 확인한 주요 동작

- `30개입`을 2회 선택하면 내부적으로 `30개입_1`, `30개입_2`가 선택상품 목록에 생성됩니다.
- 2회 선택 후 `30개입` 옵션은 `2 / 2 선택`으로 표시되고 비활성화됩니다.
- 선택상품의 수량을 변경하면 상품금액과 총상품금액이 Cafe24 계산 로직으로 갱신됩니다.
- 장바구니 클릭 시 Cafe24 장바구니 페이지로 이동합니다.
- 바로 구매 클릭 시 Cafe24 주문 또는 비회원 로그인 흐름으로 이동합니다.

## 제출 메모

본 작업은 Cafe24 기본 구매 로직을 변경하지 않고, 기존 옵션 선택 시스템 위에 커스텀 UI를 연결하는 방식으로 구현했습니다.  
따라서 실제 결제 금액, 선택상품 생성, 수량 변경, 장바구니, 바로 구매는 Cafe24 기본 기능을 따르며, 고객에게 보이는 옵션 UI와 선택 상태만 커스텀 레이어에서 관리합니다.
